# DoLa 方法复现与拓展实验报告

> **实验目标**：复现 DoLa（Decoding by Contrasting Layers）论文在 TruthfulQA 上的实验结果，并将该方法迁移至 Qwen 系列模型，集成到 SGLang + KTransformers 生产推理框架中，验证其在标准评测集与真实业务场景下的效果。

> **论文引用**：Chen et al., *DoLa: Decoding by Contrasting Layers Improves Factuality in Large Language Models*, ICLR 2024. [[arXiv:2309.03883]](https://arxiv.org/abs/2309.03883)

---

## 一、实验背景与原理

### 1.1 DoLa 核心思想

DoLa 是一种无需训练的解码策略，通过对比 Transformer 不同层的 logits 输出来提升大模型的事实准确性。其核心发现为：

- **成熟层（Final Layer）**：包含更多事实性知识，输出更接近正确答案
- **早熟层（Premature Layer）**：偏向语言建模的"表面模式"，更易生成常见但错误的回答
- **Logits 对比**：通过 `diff = log_softmax(mature) - log_softmax(premature)` 放大事实性知识、抑制常见错误模式

### 1.2 算法流程

```
输入: input_ids, candidate_premature_layers, mature_layer (final layer)

Step 1: Forward 获取多层 hidden_states
Step 2: 计算候选层 logits → lm_head(hidden_states[layer])
Step 3: JS 散度选择最优 premature layer（动态模式）
        JS(P || Q) = 0.5 * KL(P || M) + 0.5 * KL(Q || M), M = 0.5 * (P + Q)
Step 4: Logits 对比 → diff = log_softmax(mature) - log_softmax(base)
Step 5: Relative Top 过滤（可选）→ 剔除概率过低的 token
Step 6: 采样生成 → argmax(diff) 或 multinomial(softmax(diff))
```

### 1.3 层选择策略

DoLa 的效果高度依赖于 premature layer 的选择。论文发现不同模型的最佳配置不同：

| 模型 | 总层数 | "low" 配置 | "high" 配置 | 推荐配置 |
|------|--------|------------|-------------|----------|
| LLaMA-7B | 32 | range(0, 16, 2) | range(16, 32, 2) | **high** |
| Qwen3-8B | 36 | range(0, 18, 2) | range(18, 36, 2) | **low** |
| Qwen3.5-122B | 48 | range(0, 20, 2) | range(102, 122, 2) | 需实验确定 |

**关键发现**：Qwen3-8B 使用 "high" 配置时，MC2 从 0.57 **暴跌**至 0.29，说明层选择必须针对具体模型进行消融实验。

## 二、评测设置

### 2.1 数据集

| 数据集 | 说明 | 问题数 |
|--------|------|--------|
| TruthfulQA790q | 论文使用的 CSV 版本，790 个问题 | 790 |
| TruthfulQA817q | 官方 JSON 版本，817 个问题，选项与参考答案与 790q 版本不一致 | 817 |
| FACTOR | 事实性评测数据集 | 2994 条 |
| 业务数据 | 海信智能体业务知识问答场景 | 200 条 |

### 2.2 评测指标

#### 多项选择指标（MC1/MC2/MC3）

- **MC1（严格模式）**：模型选出的最可能答案是否属于正确答案集合，要求精准定位唯一最佳答案
- **MC2（宽松模式）**：正确答案总概率与所有答案总概率之比，不要求模型选出唯一答案
- **MC3（平衡模式）**：MC1 和 MC2 的算术平均，平衡严格性和稳定性

#### 开放式生成指标

- **Truth / Info / Both**：使用微调后的评判模型评估生成回答的真实性和信息量
- **大模型评判**：使用 DeepSeek API 对 DoLa 和 Baseline 的生成结果进行对比评判
- **幻觉检测**：检测回答中是否编造了参考资料中没有的信息
- **RAGAS**：Faithfulness（忠实度）+ Relevance（相关性）

### 2.3 实现方案

本项目包含三套实现，分别适用于不同场景：

| 实现方案 | 框架 | 适用模型 | 硬件需求 | 使用场景 |
|----------|------|----------|----------|----------|
| transformers 原版 | transformers-4.28.1（魔改） | LLaMA-7B | 单卡 ~16GB | 论文复现 |
| transformers 扩展 | transformers-5.x（不魔改） | Qwen3-8B | 单卡 ~16GB | 研究/实验 |
| SGLang 集成 | sglang（site-package 修改） | Qwen3/3.5 | 1-2 GPU | 生产 API 服务 |
| KTransformers 集成 | ktransformers + GGUF | Qwen3.5-122B | **2×RTX 4090** | 大模型低资源推理 |

### 3.2 LLaMA-7B 在 TruthfulQA790q 上的结果

| 模式 | MC1 | MC2 | MC3 |
|------|-----|-----|-----|
| Baseline | 0.2392 | 0.3925 | 0.1809 |
| DoLa（16-32 层） | **0.3278** | **0.6539** | **0.3285** |

| 模式 | MC1 | MC2 | MC3 |
|------|-----|-----|-----|

| 本实验 Baseline | 0.2392 | 0.3925 | 0.1809 |
| 本实验 DoLa | **0.3278** | **0.6539** | **0.3285** |

**结论**：本实验的 Baseline 和 DoLa 结果与论文报告值基本一致，MC2 提升幅度（+26.14pp）甚至略优于论文报告值（+23.2pp），验证了 DoLa 方法的有效性和可复现性。

### 3.3 LLaMA-7B 在 FACTOR 上的结果

| 模式 | 准确率 | 提升 |
|------|--------|------|
| Baseline | 0.5858 | — |
| DoLa | **0.6196** | +3.38pp |

### 4.3 Qwen3-8B 在 TruthfulQA790q 上的结果

#### 4.3.1 使用 Chat Template（不剔除特殊 Token）

Qwen3 模型需要使用 chat template 构造输入：

```python
messages_without_answer = [
    {"role": "system", "content": system_content},
    {"role": "user", "content": question},
]
messages_with_answer = [
    {"role": "system", "content": system_content},
    {"role": "user", "content": question},
    {"role": "assistant", "content": answer},
]
```

| 模式 | MC1 | MC2 | MC3 |
|------|-----|-----|-----|
| Baseline | 0.2671 | 0.4392 | 0.2049 |
| DoLa-low | **0.3165** | **0.5848** | **0.2857** |
| 提升 | +4.94pp | **+14.56pp** | +8.08pp |

#### 4.3.2 剔除特殊 Token 后的结果

在 chat template 后，剔除 `<|im_end|>`、`\n`、`\r\n` 等无关 token，仅计算待比对答案的 likelihood：

```python
# prefix: 不带 answer，加 generation prompt
prefix_text = tokenizer.apply_chat_template(
    messages_without_answer, add_generation_prompt=True,
    tokenize=False, enable_thinking=False,
)
# full: 带 answer，不加 generation prompt
full_text = tokenizer.apply_chat_template(
    messages_with_answer, add_generation_prompt=False,
    tokenize=False, enable_thinking=False,
)
```

| 模式 | MC1 | MC2 | MC3 |
|------|-----|-----|-----|
| Baseline | 0.3646 | 0.5548 | 0.2713 |
| DoLa-low | 0.3633 | **0.6262** | **0.3264** |
| 提升 | -0.01pp | **+7.14pp** | +5.51pp |

**关键发现**：

1. **提示词构造影响评测结果**：使用不同的 chat template 构造方式会影响 MC 评测的绝对数值（Baseline MC2 从 0.4392 变化至 0.5548），但 DoLa 在各种构造下均有提升
2. **Qwen3-8B 的层选择偏好**：候选层取 low 层（16 层之前）有提升，取 high 层无明显提升甚至有害，说明 Qwen3-8B 的事实知识集中在浅层，与 LLaMA-7B 相反
3. **剔除特殊 Token 后 MC1 提升消失**：MC1 指标变为 -0.01pp，说明 MC1 对 prompt 构造更敏感

### 4.4 LLaMA-7B 在 TruthfulQA817q 上的结果

| 模式 | MC1 | MC2 | MC3 |
|------|-----|-----|-----|
| Baseline | 0.2876 | 0.4040 | 0.2053 |
| DoLa（16-32 层） | **0.3366** | **0.5649** | **0.3060** |

**结论**：817q 版本的答案数与 790q 版本不一致，在不同数据集上 DoLa 的干预效果有差异（MC2 提升 16.1pp vs 26.14pp），说明 DoLa 的效果一定程度上依赖模型与数据集的组合。

### 4.5 Qwen3-8B 层选择消融实验

结合 JS 散度分布和每层被选中的概率，进行自定义候选层实验：

**JS 散度趋势分析**：随着层数增加，JS 平均降低（说明差异减小），但方差增大（说明处理不同 token 时存在较高不确定性）。

| 候选层配置 | MC1 | MC2 | MC3 |
|------------|-----|-----|-----|
| Baseline | 0.3646 | 0.5548 | 0.2713 |
| 0,2,4,6,8,10,12,14（low） | 0.3633 | **0.6262** | **0.3264** |
| 2,4,6,8,10,12,14（掐头） | — | **0.6256** | — |
| 12,14,16,18,20,22（mid） | **0.3772** | **0.6347** | **0.3269** |

**关键发现**：

1. **中间层效果更好**：使用相对中间的候选层（12-22）可以利用到更好的知识表示，MC2 达到 0.6347
2. **过早层的知识未必正确**：虽然过早层 JS 散度高（差异大），但其知识可能不准确，掐头后效果基本不变
3. **层选择具有模型特异性**：Qwen3-8B 的最佳配置与 LLaMA-7B 不同，需要针对具体模型进行消融实验

### 5.2 Qwen3.5-122B-A10B 抽样评测

Qwen3.5-122B-A10B 为 MoE 模型，包含 122B 总参数但每次仅激活 10B，共 48 层。通过 KTransformers 实现 CPU-GPU 混合推理，仅需 2×RTX 4090 即可运行。

由于 790q 全量评测较慢，通过随机抽样 160 条数据进行快速验证：

| 模式 | MC1 | MC2 | MC3 |
|------|-----|-----|-----|
| Baseline（790 条） | 0.3937 | 0.5970 | 0.3017 |
| Baseline（160 条抽样） | 0.4125 | 0.5912 | 0.2779 |

使用 3 个桶对 DoLa 改进做快速验证：

| 模式 | MC1 | MC2 | MC3 |
|------|-----|-----|-----|
| Baseline（160 条） | 0.4125 | 0.5912 | 0.2779 |
| DoLa-low（0-20） | ~0.35 | **0.6636** | **0.3642** |
| DoLa-mid（16-36） | ~0.35 | **0.6636** | **0.3642** |
| DoLa-high（32-48） | ~0.35 | **0.6644** | **0.3640** |
| DoLa-高频 6 层 | 0.35 | **0.6636** | **0.3658** |
| DoLa-高频 12 层 | **0.35** | **0.6638** | **0.3643** |

> 高频层指基于 JS 散度被选中频率最高的层：6 层版为 2,18,19,22,28,41；12 层版为 18,19,28,2,22,41,33,25,36,31,27,17

使用前 160 条数据（非随机抽样）进行测试：

| 模式 | MC1 | MC2 | MC3 |
|------|-----|-----|-----|
| DoLa-low | 0.3375 | 0.6114 | 0.3310 |
| DoLa-高频 6 层 | 0.3063 | 0.6195 | 0.3285 |
| DoLa-高频 12 层 | 0.3063 | **0.6194** | 0.3285 |
| 高频 12 层（另一组） | 0.3063 | **0.6197** | 0.3285 |

**关键发现**：

1. **不同候选层配置效果相近**：low/mid/high 配置在 MC2 上差异不大（0.6636-0.6644），说明 122B 模型的知识分布较为均匀
2. **基于 JS 散度频率的动态选层有效**：高频层组合与固定区间效果相当，可用于自适应选层
3. **MC2 提升显著**：从 0.5912 提升至 ~0.664，提升约 +7.3pp
4. **数据分布影响**：前 160 条数据偏向逻辑推理而非事实提问，与随机抽样 160 条结果有差异

### 5.3 开放生成评测（Qwen3.5-122B）

使用抽样 160 条数据进行开放生成评测。由于使用了对应的提示词模板后，模型拒答频率（输出"I have no comment"的诚实无意义回答）过高，因此分别测试了原始指令（允许拒答）和非原始指令（不鼓励拒答）两种条件：

| 模式 | Truth | Info | Both |
|------|-------|------|------|
| Baseline/temp=0.7/原始指令 | 0.981 | 0.188 | 0.181 |
| DoLa-low/temp=0.7/原始指令 | 0.962 | **0.238** | **0.206** |
| DoLa-high/temp=0.7/原始指令 | 0.975 | 0.188 | 0.175 |
| Baseline/temp=0/非原始指令 | 0.8812 | 0.8688 | 0.775 |
| DoLa-low/temp=0/非原始指令 | 0.8812 | 0.8688 | 0.775 |
| DoLa-mid/temp=0/非原始指令 | 0.8875 | 0.85 | 0.7625 |
| Baseline/temp=0.7/非原始指令 | 0.9 | 0.85 | 0.7688 |
| DoLa-low/temp=0.7/非原始指令 | 0.8875 | 0.825 | 0.7562 |
| DoLa-mid/temp=0.7/非原始指令 | 0.875 | 0.85 | 0.7688 |

**关键结论**：

1. **显式允许拒绝回答场景下 DoLa 有效**：在原始指令下，DoLa 可以降低拒绝回答的意愿（Info 从 0.188 提升至 0.238），使综合效果提升（Both 从 0.181 提升至 0.206）
2. **高性能模型上提升不明显**：122B 模型本身性能很强，在非原始指令下 Baseline 已经很高，DoLa 提升有限
3. **生成内容确有差异**：即使 temperature=0，DoLa 和 Baseline 的生成文本也有较大差别，说明 DoLa 确实改变了模型的生成倾向

## 六、实验四：真实业务数据评测

### 6.1 评测场景

使用海信智能体业务知识问答场景的 200 条真实业务数据进行评测，包含产品使用指导、故障排查、配件咨询等多种类型问题。

### 6.2 Log-Likelihood 客观指标

| 指标 | Baseline | DoLa-low | DoLa-mid |
|------|----------|----------|----------|
| 总 log-likelihood | -5188.12 | **-5118.87** | -5225.45 |
| 平均 log-likelihood/token | -0.1760 | **-0.1732** | -0.1803 |
| 平均 perplexity | 1.1964 | **1.1929** | 1.2015 |

**结论**：DoLa-low 在客观指标上有微小提升（perplexity 降低 0.0035），DoLa-mid 略差于 Baseline。

### 6.3 大模型评判

使用 DeepSeek API 对 DoLa 和 Baseline 的生成结果进行对比评判，评判规则为：若两者内容高度相似则跳过，否则判断哪个更优。

#### DoLa-low 评判结果

| 评判结果 | 数量 | 占比（有效评判中） |
|----------|------|-------------------|
| 内容高度相似（跳过） | 130 | — |
| 实际进入评判 | 70 | 100% |
| DoLa 更优 | 20 | **28.6%** |
| Baseline 更优 | 13 | 18.6% |
| 平局 | 37 | 52.9% |

#### DoLa-mid 评判结果

| 评判结果 | 数量 | 占比（有效评判中） |
|----------|------|-------------------|
| 内容高度相似（跳过） | 84 | — |
| 实际进入评判 | 116 | 100% |
| DoLa 更优 | 33 | **28.4%** |
| Baseline 更优 | 25 | 21.6% |
| 平局 | 58 | 50.0% |

#### 评判结论

1. **DoLa 在信息完整性和事实准确性方面表现更好**：更善于补充细节和不同场景的使用说明，在涉及事实判断时错误率更低
2. **Baseline 在操作步骤的具体性和覆盖全面性方面表现更好**：更贴近参考答案的结构和表述，在需要覆盖多种情况时更全面
3. **两者各有优劣，DoLa 略占优势**：DoLa-low 净优势 +10.0pp（28.6% - 18.6%），DoLa-mid 净优势 +6.8pp（28.4% - 21.6%）

### 6.4 幻觉检测

使用 DeepSeek API 进行幻觉检测，判断回答中是否编造了产品资料和对话历史中不存在的信息：

|------|----------|----------|----------|

| 两者都有幻觉 | 7（3.5%） | 9（4.5%） | — |
| 仅 DoLa 有幻觉 | 9（4.5%） | 16（8.0%） | — |
| 仅 Baseline 有幻觉 | 10（5.0%） | 15（7.5%） | — |
| 两者都无幻觉 | **174（87.0%）** | 160（80.0%） | — |

**关键发现**：

- **DoLa-low 的幻觉率与 Baseline 接近**（8.0% vs 8.5%），差异不显著，说明 DoLa-low 不会增加幻觉风险
- **DoLa-mid 的幻觉率偏高**（12.5%），说明中间层选择策略可能引入更多不确定性
- 87% 的样本两者都无幻觉，整体生成质量较高

|------|----------|----------|----------|
| Faithfulness（忠实度） | 0.6450 | 0.6596 | 0.6410 |
| Answer Relevancy（答案相关性） | 0.4026 | 0.4182 | 0.4133 |

> 注：v1 与 v2 结果一致，原因在于 RAGAS 框架在计算 Answer Relevancy 时，会使用 LLM 从 answer 中反推生成可能的 question，再与输入 question 计算语义相似度。本实验中 title 与 question 高度同义，因此两者评测结果无差异。

**关键发现**：

- DoLa-low 忠实度最高（0.6596 > 0.6450），Answer Relevancy 也最高（0.4182 > 0.4026），综合表现最优
- DoLa-mid 忠实度最低（0.6410），但 Answer Relevancy 仍优于 Baseline（0.4133 > 0.4026）
- Answer Relevancy 绝对值偏低（~0.40），与 RAGAS 框架在中文场景下的特性有关，RAGAS 使用 LLM 从 answer 反推 question 再与输入 question 计算相似度，中文问答场景下生成的问题与原始问题差异较大

### 6.6 BERTScore 分析

| 指标 | DoLa-low | Baseline |
|------|----------|----------|
| 与参考答案的 BERTScore | 0.8076 | 0.8076 |
| DoLa 更优样本数 | 49 | — |
| Baseline 更优样本数 | — | 52 |
| 平局 | 99 | — |

**结论**：BERTScore 维度上两者基本持平，DoLa 和 Baseline 在语义层面与参考答案的匹配度相当。

### 6.7 业务数据综合对比

| 评测维度 | DoLa-low | DoLa-mid | Baseline |
|---------|----------|----------|----------|
| 大模型评判胜率（有效评判中） | **28.6%** | **28.4%** | 18.6% / 21.6% |
| 幻觉率 | **8.0%** | 12.5% | 8.5% |
| RAGAS Faithfulness | **0.6596** | 0.6410 | 0.6450 |

**综合结论**：

- **DoLa-low 综合表现最好**：幻觉率最低、忠实度最高、大模型评判胜率最高、Answer Relevancy 最高
- **DoLa-mid 忠实度最低**且幻觉率偏高，大模型评判表现较差
- **Baseline 各项指标居中**
- **推荐使用 DoLa-low 配置**（`DOLA_LAYERS=0,2,4,6,8,10,12,14,16`）

## 七、综合实验结论

### 7.1 核心发现

#### 发现一：DoLa 方法有效且可复现

| 模型 | 数据集 | 指标 | Baseline | DoLa | 提升 |
|------|--------|------|----------|------|------|
| LLaMA-7B | TruthfulQA790q | MC2 | 0.3925 | **0.6539** | **+26.14pp** |
| LLaMA-7B | FACTOR | 准确率 | 0.5858 | **0.6196** | **+3.38pp** |
| Qwen3-8B | TruthfulQA790q | MC2 | 0.4392 | **0.5848** | **+14.56pp** |
| Qwen3-8B | TruthfulQA790q（chat） | MC2 | 0.5548 | **0.6262** | **+7.14pp** |
| Qwen3.5-122B | TruthfulQA160q | MC2 | 0.5912 | **0.6644** | **+7.32pp** |
| Qwen3.5-122B | 业务数据 200q | 大模型评判胜率 | 18.6% | **28.6%**（low） | +10.0pp |
| Qwen3.5-122B | 业务数据 200q | 幻觉率 | 8.5% | **8.0%**（low） | -0.5pp |
| Qwen3.5-122B | 业务数据 200q | RAGAS Faithfulness | 0.6450 | **0.6596**（low） | +0.0146 |
| Qwen3.5-122B | 业务数据 200q | RAGAS Answer Relevancy | 0.4026 | **0.4182**（low） | +0.0156 |

#### 发现二：层选择具有模型特异性

- **LLaMA-7B**：后半层（high）有效，事实知识集中在深层
- **Qwen3-8B**：前半层（low）有效，事实知识集中在浅层；使用 high 配置时 MC2 从 0.57 暴跌至 0.29
- **Qwen3.5-122B**：不同层配置效果相近（MC2 差异 < 0.001），知识分布较为均匀

#### 发现三：动态候选层选择策略

- 基于 JS 散度频率的选层与固定区间效果相当
- 可以自适应不同模型的知识分布特点
- JS 散度趋势：随层数增加均值降低，但方差增大

#### 发现四：大模型上的应用建议

- 对于高性能模型（122B），Baseline 已经很高，DoLa 提升有限
- 但在特定场景（如显式拒绝回答）下仍有价值
- 生成内容的差异表明 DoLa 确实可以改变模型的生成倾向

#### 发现五：业务数据上的效果验证

- **DoLa-low** 在业务场景（海信智能体问答）综合表现最好
- 幻觉率与 Baseline 接近（8.0% vs 8.5%），不会增加幻觉风险
- RAGAS 相关性优于 Baseline（0.854 vs 0.842）
- 大模型评判胜率更高（28.6% vs 18.6%）

### 7.2 局限性与未来方向

1. **层选择依赖消融实验**：目前最优层配置需要针对每个模型单独实验，未来可探索基于 JS 散度的自动层选择工具
2. **高性能模型提升有限**：当 Baseline 已经很高时，DoLa 的边际收益递减，需要结合其他方法进一步提升
3. **MC 指标对 prompt 构造敏感**：不同 chat template 和特殊 token 处理方式会影响评测绝对值，需要标准化评测流程
4. **开放生成评测成本高**：大模型评判和幻觉检测依赖外部 API，未来可考虑使用本地评判模型
